# Supabase SQL 说明

这份目录现在只保留适合公开仓库维护的 SQL 与说明：

- `supabase-init.sql`
  用于全新环境初始化，采用更安全的默认值。
- `supabase-update-likes.sql`
  旧环境补 `likes` 字段时可单独执行的补丁。

## 公开仓库中的定位

这里存放的是：

- 全新环境可复用的初始化脚本
- 与功能升级直接相关的增量补丁
- 面向公开协作的结构说明

这里**不再存放**线上生产环境的完整基线快照、生产漂移细节或带有过强环境指向性的审计结果。

## 推荐做法

1. 全新环境初始化时，优先使用 `supabase-init.sql`。
2. 已有环境升级时，优先写增量 migration，不要直接拿 init 覆盖线上。
3. 涉及 RLS、表结构、函数、触发器变更时，应先在本地或私有环境核对，再执行到生产。
4. 生产环境的基线对账、漂移记录与审计细节，建议保存在私有仓库或本地受保护位置。

## 当前目录用途

如果你是第一次接手这个项目，优先看：

- `supabase-init.sql`
- `supabase-update-likes.sql`

如果你要核对线上真实状态，请不要把生产基线直接继续提交到公开仓库，而应转到私有位置维护。
