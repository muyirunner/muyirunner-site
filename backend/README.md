# backend/ - 历史方案与备用设计说明

> 说明：这个目录当前**不是网站生产主链路**。
>
> 目前站点公开运行的博客与内容管理，主用的是 **Supabase 动态方案**。
> `backend/` 中保留的内容主要用于：
>
> - 历史设计归档
> - 备用自建后端方案记录
> - Directus / Express / Prisma 等路线的实验性草稿

## 当前状态

这个目录里的文档与脚本可以作为：

- 架构参考
- 历史方案备忘
- 后续自建 CMS / 自建 API 时的素材

但它们**不应被误认为当前线上生产环境正在使用的唯一事实来源**。

## 阅读建议

如果你想了解当前生产主链路，优先查看：

- 仓库根目录 `README.md`
- `src/lib/supabase.ts`
- `database/supabase/`

如果你想研究备用路线或历史方案，再查看本目录中的文档。

## 安全说明

本目录中的示例配置、默认密码、数据库连接示意等内容均应视作**占位示例**，不要直接用于生产环境。

建议统一替换成如下形式：

- `your-admin-password`
- `your-database-url`
- `your-secret-key`

## 目录定位

你可以把 `backend/` 理解为：

- **archive / experiments / drafts**

而不是当前线上网站的唯一后端实现。
