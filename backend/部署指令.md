# 🚀 Directus CMS 一键部署指令

由于服务器网络响应较慢，我已经为你准备好了所有脚本。请按照以下步骤操作：

## 📦 第一步：上传部署脚本

在你的**本地终端**（Windows PowerShell 或 CMD）中执行：

```bash
# 进入项目目录
cd "d:\代码合集创建的各种代码放于此处\my-site"

# 上传快速部署脚本
scp backend/quick-deploy.sh admin@muyirunner.icu:/home/admin/
scp backend/configure-nginx.sh admin@muyirunner.icu:/home/admin/
```

## 🔧 第二步：在服务器上执行部署

SSH 登录到服务器：

```bash
ssh admin@muyirunner.icu
```

在服务器上执行：

```bash
# 给脚本执行权限
chmod +x /home/admin/quick-deploy.sh
chmod +x /home/admin/configure-nginx.sh

# 运行部署（这步需要 5-10 分钟）
/home/admin/quick-deploy.sh
```

**重要**：部署过程中可能会看到：
- `npm install` 进度条可能停顿，这是正常的，请耐心等待
- 如果提示需要 sudo 密码，请输入你的密码

## ⚙️ 第三步：配置 Nginx

部署完成后，配置 Nginx：

```bash
# 方式 1：手动编辑（推荐）
sudo nano /etc/nginx/sites-available/muyirunner.icu

# 在 server { ... } 块中，找到现有的 location 块
# 在最后一个 location 块之后，添加以下内容：

    # CMS 管理后台
    location /admin {
        proxy_pass http://localhost:8055;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }

    # CMS API
    location /api/cms/ {
        rewrite ^/api/cms/(.*)$ /$1 break;
        proxy_pass http://localhost:8055;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # CMS 上传文件
    location /cms-uploads/ {
        alias /home/admin/blog-cms/uploads/;
        access_log off;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

# 保存文件（Ctrl+O, Enter, Ctrl+X）

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl reload nginx
```

## ✅ 第四步：验证部署

```bash
# 检查 Directus 状态
pm2 status

# 应该看到 blog-cms 状态为 "online"

# 查看日志
pm2 logs blog-cms --lines 20

# 测试本地访问
curl http://localhost:8055/server/health

# 应该返回 {"status":"ok"}
```

## 🆘 遇到问题？

### 问题 1：npm install 一直卡住

```bash
# 按 Ctrl+C 停止
# 切换到国内镜像
npm config set registry https://registry.npmmirror.com

# 重新运行部署脚本
/home/admin/quick-deploy.sh
```

### 问题 2：PM2 启动失败

```bash
# 查看详细错误
pm2 logs blog-cms --err

# 手动启动测试
cd /home/admin/blog-cms
npx directus start
```

### 问题 3：访问 /admin 返回 502

```bash
# 检查 Directus 是否运行
pm2 status

# 检查端口监听
sudo netstat -tulpn | grep 8055

# 检查 Nginx 配置
sudo nginx -t
```
